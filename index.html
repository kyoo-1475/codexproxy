Enterprise-grade Pool Mode

Pool Mode adds intelligent routing with failover, multi-key rotation, cooldowns, and curated logical profiles. Ship resilient AI workflows without managing complexity.

Failover

Retry another provider when one fails.

Multi-key rotation

Spread traffic across multiple API keys.

Cooldowns

Pause unhealthy providers automatically.

Logical profiles

Expose fast, balanced, and strong models.

Available Profiles
codex-fast
Low latency. Optimized for speed and completions.
codex-balanced
Best overall quality, cost, and latency. Default.
codex-strong
Max quality for complex architectural tasks.

pool.yaml
pools:
  codex-fast:
    strategy: priority
    cooldown: 60
    retries: 2
    providers:
      - openrouter: { api_key: ${KEY}, priority: 1 }
      - deepseek:   { api_key: ${KEY}, priority: 2 }

  codex-balanced:
    strategy: weighted
    providers:
      - nvidia:     { api_key: ${KEY}, weight: 3 }
      - openrouter: { api_key: ${KEY}, weight: 1 }

  codex-strong:
    strategy: priority
    providers:
      - nvidia:     { best_effort: true, priority: 1 }
      - openrouter: { api_key: ${KEY}, priority: 2 }

Quick Start

Get up and running in less than 60 seconds.

1. Clone & Enter
$git clone https://github.com/kyoo-147/codex_proxy_for_all_models.git
cd codex_proxy_for_all_models
2. Configure
$export CODEX_PROXY_UPSTREAM_BASE_URL=http://127.0.0.1:11434/v1
export CODEX_PROXY_UPSTREAM_MODEL=llama3.1:70b
3. Run Proxy
$python -m codex_proxy_for_all_models

Deployment Information
Docker support can be added with a small image, but this repo is designed to run directly with Python.
~/.codex/config.toml
model = "llama3.1:70b"
model_provider = "local_model_proxy"

name = "Local Model Proxy"
base_url = "http://127.0.0.1:8787"
